Selection Sort Algorithm (C++)

#include <iostream>
using namespace std;

int main() {
    // Taking Inputs
    int n;
    cin >> n;

    int arr[n];
    for(int i = 0; i < n; i++) {
        cin >> arr[i];
    }

    // Selection Sort Algorithm
    for(int i = 0; i < n - 1; i++) {
        int minIndex = i;

        for(int j = i + 1; j < n; j++) {
            if(arr[j] < arr[minIndex]) {
                minIndex = j;
            }
        }

        int temp = arr[i];
        arr[i] = arr[minIndex];
        arr[minIndex] = temp;
    }

    // Output the sorted array
    for(int i = 0; i < n; i++) {
        cout << arr[i] << " ";
    }

    cout << "\n";

    return 0;
}

Selection Sort is a simple sorting algorithm that works by repeatedly finding the smallest (or largest) element from the unsorted portion of the array and swapping it with the leftmost unsorted element. This process divides the array into two parts: one that is sorted and the other that remains unsorted. The algorithm continues this process until the entire array is sorted. Although easy to understand and implement, Selection Sort is not the most efficient for large datasets due to its time complexity of O(n²), which makes it slower as the number of elements increases. It performs well with small arrays but becomes impractical when dealing with larger ones.

One of the advantages of Selection Sort is that it is an in-place sorting algorithm, meaning it doesn’t require additional memory beyond the array being sorted, giving it a space complexity of O(1). Despite its inefficiency in terms of time complexity, Selection Sort can still be useful in certain situations, particularly when memory usage is a concern or when working with small datasets where the quadratic time complexity doesn’t significantly impact performance. However, for larger datasets, more efficient algorithms like Quick Sort or Merge Sort, which have time complexities of O(n log n), are typically preferred.
